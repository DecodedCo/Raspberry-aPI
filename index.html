Our brief

Store check-in data

Our brief requires us to store some data about checkins. We can't just store this in our user's browser because data stored in the browser is temporary. If the user were to close their browser, delete their cookies or access your app from a different device, there would be no record that they had previously checked in.

We want a common data store; a central repository of all user checkins. So, we need to store our data on a single computer that we can control — a web server.

Our server

A $25 computer

A server might be provided by someone like Amazon Web Services, but we chose a Raspberry Pi — a credit card-sized computer that you can buy for $25. It's sitting in the corner of our London HQ!

We chose a Raspberry Pi to demonstrate that there's nothing magical about a web-server. In the real world, you want to use something a bit more sturdy. If someone spills tea on the Pi, or unplugs it, or our internet connection breaks â€“ the server will stop working!

Don't worry, we have a back-up at: form.decoded.co:3000

Our data

Name and number

Your app will ask users to input their username to check-in. It will then display their username and the number of times they have checked in. We need to store:

  1. Name of user
  2. Total number of user's checkins

The data is stored in JSON format – take a look at the example below:

{
	decodedco: 1,
	timbernerslee: 5,
	adalovelace: 3,
	turing404: 2
}

Our instructions

Store and access

We have created this API to make it really easy to store and access data on our server.

Each workshop will have a unique name assigned to it, usually the date of the session (e.g. 01-04-13). That data will be accessible at the following URL:

http://api.decoded.co/checkin/01-04-13

The first time someone checks into one of the apps created during your session a new datastore will automatically be created - this initial datastore will then be used to store the data for everyone's app.

To make a checkin, you can append a query to this URL using the "username" parameter. For example:

http://api.decoded.co/checkin/01-04-13?username=adalovelace

Example usage

Forms and AJAX

1) Form

The simplest way is to use an HTML form with a single input. To choose where to send the data, you will need to set the form's "action" attribute to your session's API URL. Your form should include a single text input box with the "name" attribute set to "username". For example:

<form action="http://api.decoded.co/checkin/04-07-13">
	<input type="text" name="username">
</form>

2) AJAX

You can also submit data via AJAX. AJAX can send data to and retrieve data from the server without leaving the current page. We recommend using jQuery's AJAX function, as follows:

$.ajax({
url: 'http://api.decoded.co/checkin/04-07-13',
	data: { username: "turing404" },
    success: function(result){
		console.log(result);
	} // END success
}); // END ajax

For more information, please read the jQuery .ajax() documentation.
