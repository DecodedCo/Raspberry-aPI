Decoded API Documentation

What is Decodeds API?

To help you build your Code in a Day app, Decoded has created an API that allows you to store data on a server. api.decoded.co is actually hosted on a Raspberry Pi, a credit card-sized computer/server? that you can buy for $25.

With a few lines of code, your app can send data to this server to be stored. It can also pull data down from the server to be displayed in your app.

What data is stored in the API?

Your app asks users to input their Twitter username and press return, it then displays their username and the number of times they have checked in. For this to work, we need to store a list of usernames with a corresponding number of checkins.

How is the data stored in the API?

The data is stored in a JSON format, take a look at the example below.

{
    decodedco: 1, 
    johnridpath: 2,
    kathrynparsons: 3,
    aliblackwell: 4
}

Where is the data stored in the API?

Each workshop will have a unique name assigned to it, usually the date of the session (e.g. 01-04-13). That data will be accessible at the following URL: http://api.decoded.co/checkin/01-04-13 The first time someone checks into one of the apps created during your session a new datastore will automatically be created - this initial datastore will then be used to store the data for everyone's apps. To make a checkin, you can append a query to this URL using the "username" parameter. For example: http://api.decoded.co/checkin/01-04-13?username=kathrynparsons

Submitting data from your app

You have a few options as to how you might submit data to a datastore in this api.

2) AJAX

You can also submit data via AJAX. AJAX can send data to and retrieve data from the server without leaving the current page. We recommend using jQuery's AJAX function, as follows:

$.ajax({
    url: 'http://api.decoded.co/checkin/04-07-13',
    data: {username: "DecodedCo"},
    method: 'POST',
    dataType: 'jsonp',
    jsonp: 'callback',
    jsonpCallback: 'jsonpCallback',
    success: function(data){
        console.log(data);
    }
});

For more information, please read the jQuery .ajax() documentation (http://api.jquery.com/jQuery.ajax/).
